## WantedTextArea

`WantedTextArea`는 여러 줄 텍스트 입력이 필요한 경우 사용하는 커스텀 입력 필드 컴포넌트입니다. `String` 또는 `TextFieldValue`를 통해 상태를 제어할 수 있으며, 우측 버튼, 상태 메시지, 아이콘, 문자 수 카운터 등의 기능을 제공합니다.

### 사용 예시
```kotlin
WantedTextArea(
    text = "내용",
    title = "자기소개",
    placeholder = "자유롭게 작성해주세요",
    rightButton = "저장",
    onValueChange = { text -> ... }
)
```

### 파라미터

### WantedTextArea (String 기반)
| 이름 | 타입 | 설명 |
|:---|:---|:---|
| text | `String` | 입력된 텍스트입니다. |
| modifier | `Modifier` | 외형 및 레이아웃 설정입니다. |
| placeholder | `String` | 플레이스홀더 텍스트입니다. |
| title | `String` | 상단 제목 텍스트입니다. |
| description | `String?` | 하단 설명 또는 메시지입니다. |
| rightButton | `String?` | 우측 버튼 텍스트입니다. |
| leadingContent | `(() -> Unit)?` | 왼쪽 슬롯 콘텐츠입니다. |
| trailingContent | `(() -> Unit)?` | 오른쪽 슬롯 콘텐츠입니다. |
| enabled | `Boolean` | 입력 가능 여부입니다. 기본값은 true입니다. |
| negative | `Boolean` | 에러 상태 여부입니다. |
| maxLines | `Int` | 최대 줄 수입니다. 기본값은 3입니다. |
| minLines | `Int` | 최소 줄 수입니다. 기본값은 1입니다. |
| maxWordCount | `Int` | 최대 입력 글자 수입니다. 기본값은 2000자입니다. |
| enabledOverflowText | `Boolean` | 글자 수 초과 허용 여부입니다. 기본값은 false입니다. |
| isGraphemeClusterCount | `Boolean` | 글자 수를 grapheme cluster 기준으로 계산할지 여부입니다. |
| requiredBadge | `Boolean` | 필수 입력 뱃지 표시 여부입니다. |
| interactionSource | `MutableInteractionSource` | 포커스 및 상태 추적용입니다. |
| focused | `State<Boolean>` | 포커스 상태입니다. |
| keyboardOptions | `KeyboardOptions` | 키보드 동작 설정입니다. |
| keyboardActions | `KeyboardActions` | 키보드 액션 처리입니다. |
| background | `Color` | 배경 색상입니다. |
| onClickRightButton | `() -> Unit` | 우측 버튼 클릭 콜백입니다. |
| onValueChange | `(String) -> Unit` | 텍스트 변경 시 호출됩니다. |

## Note
- `isGraphemeClusterCount = true`인 경우 이모지 등을 포함한 실제 사용자 인식 단위로 글자 수를 계산합니다.
- `enabledOverflowText = false`인 경우 `maxWordCount`를 초과한 입력은 무시됩니다.
- 우측 버튼은 에러 상태일 경우 아이콘으로 대체될 수 있습니다.
- `WantedTextAreaCharacterCount`를 통해 실시간 문자 수 카운트를 표시할 수 있습니다.
