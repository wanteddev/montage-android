## String.accent
문자열 내에서 특정 부분 텍스트를 강조하여 AnnotatedString을 생성합니다.

전체 문자열에서 subText를 찾아 지정된 textStyle을 적용합니다. 동일한 subText가 여러 번 나타나면 모두 강조됩니다.

```kotlin
"안녕하세요 홍길동님".accent(
    subText = "홍길동",
    textStyle = TextStyle(color = Color.Blue)
)
```

### Parameters
| 이름 | 타입 | 설명 |
|:---|:---|:---|
| subText | String | 강조할 부분 텍스트입니다. |
| textStyle | TextStyle | 강조 부분에 적용할 텍스트 스타일입니다. |

### Return
| 타입 | 설명 |
|:---|:---|
| AnnotatedString | 강조 스타일이 적용된 AnnotatedString을 반환합니다. |

## Overload
문자열 내에서 여러 부분 텍스트를 강조하여 AnnotatedString을 생성합니다.

전체 문자열에서 여러 개의 subText를 찾아 지정된 textStyle을 적용합니다. 각 subText가 여러 번 나타나면 모두 강조됩니다.

```kotlin
"안녕하세요 홍길동님, 반갑습니다 홍길동님".accent(
    "홍길동", "반갑습니다",
    textStyle = TextStyle(color = Color.Blue, fontWeight = FontWeight.Bold)
)
```
<details>
  <summary>Parameters</summary>
  
  | 이름 | 타입 | 설명 |
  |:---|:---|:---|
  | subText | vararg String | 강조할 여러 부분 텍스트들입니다. |
  | textStyle | TextStyle | 강조 부분에 적용할 텍스트 스타일입니다. |
</details>

### Return
| 타입 | 설명 |
|:---|:---|
| AnnotatedString | 강조 스타일이 적용된 AnnotatedString을 반환합니다. |

## Advanced
### Modifier.getBorderModifier
컴포넌트에 테두리 스타일을 적용하는 Modifier 확장 함수입니다.

원형 또는 둥근 모서리 형태에 대해 테두리 없음, 외곽선, 내부선 타입을 지원합니다. 테두리 타입에 따라 적절한 Modifier 체인을 반환합니다.

```kotlin
Modifier.getBorderModifier(
    size = 40.dp,
    isCircleShape = true,
    borderType = BorderType.OutLine,
    borderWidth = 2.dp,
    borderColor = Color.Gray
)
```

#### Parameters
| 이름 | 타입 | 설명 |
|:---|:---|:---|
| size | Dp | 컴포넌트의 크기입니다. |
| isCircleShape | Boolean | 원형 여부입니다. true면 원형, false면 둥근 모서리입니다. |
| borderType | BorderType | 테두리 타입입니다 (None, OutLine, InnerLine). |
| cornerRadius | Dp | 모서리 반경입니다. 기본값은 0.dp입니다. |
| borderWidth | Dp | 테두리 두께입니다. 기본값은 1.dp입니다. |
| borderColor | Color | 테두리 색상입니다. |
| backgroundColor | Color | 배경 색상입니다. |

#### Return
| 타입 | 설명 |
|:---|:---|
| Modifier | 테두리 스타일이 적용된 Modifier를 반환합니다. |

## Enum
### BorderType
컴포넌트의 테두리 스타일 타입을 정의하는 enum 클래스입니다.

| 값 | 설명 |
|:---|:---|
| None | 테두리가 없는 형태입니다. |
| InnerLine | 내부에 그려지는 테두리입니다. |
| OutLine | 외부로 확장되는 테두리입니다. |
